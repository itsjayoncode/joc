# 002 Development Roadmap

## Why This Document Exists

This document gives contributors a sequencing model for how JOC should grow so new work aligns with the active repository phase.

## Planned Order of Work

1. Repository Foundation
2. Developer Tooling
3. Engineering Automation
4. Developer Experience Platform
5. Release Engineering
6. Package Engineering Standards
7. First Public Package Release

## Sequencing Rationale

- Foundation comes first so future tooling has stable ground.
- Tooling comes before package implementation so quality expectations are clear.
- Documentation and playground support package development and public understanding.
- Release engineering should arrive before the first package implementation so package publication rules are already clear.

## Expected Near-Term Focus

The next major repository investments should center on package engineering standards and the first package implementation blueprint.
