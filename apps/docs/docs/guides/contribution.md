# Contribution

JOC welcomes early contributions, especially in documentation, architecture, and repository health.

## Good contributions in this phase

- improving clarity in docs
- refining information architecture
- strengthening build, lint, or test reliability
- improving contributor onboarding

## What not to do in this phase

- implement package features outside the active roadmap
- add release or publishing logic early
- introduce new tools that the roadmap has not justified yet

For repository-wide expectations, also read the root [CONTRIBUTING.md](https://github.com/JayOnCode/joc/blob/main/CONTRIBUTING.md).
