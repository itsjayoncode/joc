# Common Patterns

Production-ready patterns built on Browser Lifecycle.

| Pattern                                                  | Use case                             |
| -------------------------------------------------------- | ------------------------------------ |
| [Auto Save](/packages/browser-lifecycle/patterns/auto-save)                         | Persist drafts when the user pauses  |
| [Analytics](/packages/browser-lifecycle/patterns/analytics)                         | Emit presence-aware analytics events |
| [Presence Detection](/packages/browser-lifecycle/patterns/presence-detection)       | Track active vs idle users           |
| [Session Timeout](/packages/browser-lifecycle/patterns/session-timeout)             | Warn before session expiry           |
| [Background Sync](/packages/browser-lifecycle/patterns/background-sync)             | Sync when connectivity returns       |
| [Offline First](/packages/browser-lifecycle/patterns/offline-first)                 | Queue work while offline             |
| [Polling](/packages/browser-lifecycle/patterns/polling)                             | Pause polling when hidden            |
| [Page Tracking](/packages/browser-lifecycle/patterns/page-tracking)                 | Track page visibility sessions       |
| [Visibility Pause](/packages/browser-lifecycle/patterns/visibility-pause)           | Pause expensive work when hidden     |
| [Focus Resume](/packages/browser-lifecycle/patterns/focus-resume)                   | Resume on window focus               |
| [Shared WebSocket](/packages/browser-lifecycle/patterns/shared-websocket)           | One socket in the primary tab        |
| [Leader Election](/packages/browser-lifecycle/patterns/leader-election)             | Elect a single coordinator tab       |
| [Plugin Architecture](/packages/browser-lifecycle/patterns/plugin-architecture)     | Extend runtime behavior safely       |
| [State Synchronization](/packages/browser-lifecycle/patterns/state-synchronization) | Share state across tabs              |

Each pattern links to playground pages and framework examples where applicable.
